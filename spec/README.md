# Specification

This directory contains the Open Specification documents for your project.

**Your first step:** Fill in these four documents to define what you're building.

## Documents

| File | Purpose | Order |
|------|---------|-------|
| [PRODUCT.md](PRODUCT.md) | What we're building and why | 1st — Start here |
| [ROADMAP.md](ROADMAP.md) | Phased construction plan | 2nd |
| [ARCHITECTURE.md](ARCHITECTURE.md) | System structure and design decisions | 3rd |
| [SPECIFICATION.md](SPECIFICATION.md) | Detailed functional requirements | 4th |

Each document contains guidance on what to write. Open the file, read the comment
block, then replace it with your specification.

## How These Are Used

Per [OPENSPEC.md](../OPENSPEC.md), these documents are the source of truth for what
your project should be. All implementation work is driven by GitHub Issues that
reference specific sections of these documents.

The spec evolves as you learn. After each phase, update documents to reflect new
understanding, changed requirements, or refined architecture.
