# Specification

This directory contains the Open Specification for the Open Specification Template
for Python. These documents define what the template is, how it's structured, what
it must do, and in what order it should be built.

## Documents

| File | Purpose | Status |
|------|---------|--------|
| [PRODUCT.md](PRODUCT.md) | What we're building and why | v0.1.0 |
| [ROADMAP.md](ROADMAP.md) | Phased construction plan | v0.1.0 |
| [ARCHITECTURE.md](ARCHITECTURE.md) | System structure and design decisions | v0.1.0 |
| [SPECIFICATION.md](SPECIFICATION.md) | Detailed functional requirements | v0.1.0 |

## How These Are Used

Per [OPENSPEC.md](../OPENSPEC.md), these documents are the source of truth for what
the template should be. All implementation work is driven by GitHub Issues that
reference specific sections of these documents.

The spec evolves as we learn. After each phase, documents are updated to reflect
new understanding, changed requirements, or refined architecture.
